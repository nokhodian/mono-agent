# Image Vault — Design Spec
**Date:** 2026-04-09  
**Status:** Approved

---

## Overview

A central image store for the Mono Agent Wails app. All images generated by workflows (Gemini, HuggingFace, etc.) are automatically registered here. Users can also add images manually via drag-drop or file picker. Images are referenced in workflow configs using `@img-001` syntax which resolves to the absolute file path at execution time.

---

## Data Layer

### Migration: `data/migrations/009_image_vault.sql`

```sql
CREATE TABLE vault_images (
    id           TEXT PRIMARY KEY,       -- "img-001", "img-002", ...
    seq          INTEGER NOT NULL UNIQUE, -- numeric part, used for ordering and @-refs
    path         TEXT NOT NULL,           -- ~/.monoes/vault/img-001.png
    filename     TEXT NOT NULL,           -- original filename for display
    size_bytes   INTEGER NOT NULL DEFAULT 0,
    source       TEXT NOT NULL DEFAULT 'upload', -- 'gemini' | 'upload' | 'huggingface'
    workflow_id  TEXT,                    -- nullable FK to workflows
    execution_id TEXT,                    -- nullable FK to workflow_executions
    label        TEXT,                    -- optional user-set name
    created_at   TIMESTAMP NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_vault_images_seq ON vault_images(seq DESC);
```

### File Storage

- Directory: `~/.monoes/vault/`
- Files renamed on registration to match ID: `img-001.png`, `img-002.jpg`, etc.
- `~/.monoes/downloads/` remains the Gemini staging area; images are moved to vault immediately after download.

### ID Generation

On every insert: `SELECT COALESCE(MAX(seq), 0) + 1` → format as `img-%03d`. IDs are ever-increasing and never reused.

## CLI Side — `internal/vault/` Package

New thin package (~100 lines):

```go
// Register copies src file to ~/.monoes/vault/, inserts DB row, returns vault ID.
func Register(ctx context.Context, db *sql.DB, src, source, workflowID, executionID string) (string, error)

// Resolve turns "@img-001" → "/Users/user/.monoes/vault/img-001.png"
func Resolve(db *sql.DB, ref string) (string, error)

// ResolveConfig walks a config map and replaces @img-NNN strings with absolute paths.
func ResolveConfig(db *sql.DB, config map[string]interface{}) error
```

### Gemini Bot Auto-registration

In `internal/bot/gemini/bot.go`, after `extract_and_download_images` writes a file to `~/.monoes/downloads/`:
1. Call `vault.Register()` to copy to `~/.monoes/vault/` and insert DB row with `source = "gemini"`.
2. Add `"vault_id": "img-NNN"` to the image result map so downstream nodes can reference it.
3. The `execution_id` is passed via context from the workflow engine.

### `@img-NNN` Resolution in Workflows

`vault.ResolveConfig()` is called in `internal/workflow/execution.go` after template resolution and before node execution. Any config field containing `@img-NNN` is replaced with the absolute file path. Works for all node types automatically with no per-node changes.

## Wails Backend — `wails-app/app.go`

Six new bound methods (all lightweight DB reads/writes):

| Method | Description |
|---|---|
| `GetVaultImages(limit int)` | All images, ordered by seq DESC |
| `GetVaultImage(id string)` | Single image record by ID |
| `AddVaultImage(srcPath, label string)` | Copy file to vault, register in DB |
| `DeleteVaultImage(id string)` | Delete DB record + file on disk |
| `SearchVaultImages(query string)` | Filter by label/source/workflow substring |
| `GetVaultStats()` | Total count and total size_bytes for header |

Each returned record includes: `id`, `seq`, `path`, `filename`, `size_bytes`, `source`, `workflow_id`, `execution_id`, `label`, `created_at`, and a Wails asset `url` for direct image rendering in the frontend.

### `ImageVault.jsx` — Main Page

**Header bar:** title, `23 images · 48.2 MB` stats, search input, `+ Add Image` button.

**Drop zone hint:** subtle banner below header — "Drop images anywhere to add them to the vault." Active drop target highlighted on drag-over.

**Column headers:** thumbnail | ID | Label/Filename | Source | Workflow | Size | Date | (delete)

**Rows:** 36×36px thumbnail preview, `img-NNN` ID in cyan, label (or filename if no label), source badge (`gemini` purple / `upload` green), workflow name, size, date, red 🗑 delete button.

**Source badges:** `gemini` (purple), `upload` (green), `huggingface` (blue), etc.

### Detail Modal — click any row

Centered modal, dimmed background. Contents:
- Title: `img-NNN · Label`
- Full image preview (constrained to modal width)
- Metadata row: source, size, date, workflow name
- Action buttons: `📋 Copy @img-NNN` | `✏️ Rename` | `🗑 Delete`

Clicking outside or `✕` closes the modal.

### Picker Modal — `🖼` button in workflow config fields

Opens when user clicks the image picker icon next to any image-type config field in NodeRunner. Contains:
- Search input
- Scrollable list of vault images (thumbnail + ID + label)
- `Select` button inserts `@img-NNN` into the field; `Cancel` closes

### `@`-Autocomplete in Config Fields

When user types `@` in any text config field in the workflow editor:
- A dropdown appears below the field showing matching vault images filtered by the typed suffix
- Each row: thumbnail emoji + `@img-NNN` + label
- Arrow keys to navigate, Enter to insert, Escape to dismiss
- Selecting an image inserts `@img-NNN` at the cursor position

## Error Handling

- **Missing file on load:** If a vault image's file no longer exists on disk, show a broken-image placeholder in the list. Do not crash.
- **Delete confirms inline:** Clicking 🗑 in the list deletes immediately (no confirmation dialog — too disruptive for bulk cleanup). The modal delete button does show a brief "Are you sure?" inline toggle before proceeding.
- **Upload failures:** If `AddVaultImage` fails (bad file, disk full), show an inline error banner at the top of the vault page.
- **`@img-NNN` resolution failure:** If a referenced image no longer exists at runtime, the workflow node receives the raw `@img-NNN` string and logs a warning — it does not crash the execution.

---

## Out of Scope

- Cloud storage / sync
- Image editing or transformation
- Collections / folders / tags (flat list is sufficient for v1)
- Duplicate detection
- Thumbnail generation for non-image files
